# Contributing to Proxuma IT

Proxuma follows one product rule:

> One purpose, one control, one place.

Before proposing a change:

1. Describe the user problem.
2. Identify any existing control serving the same purpose.
3. Explain the evidence or test supporting the change.
4. Document failure and empty states.
5. Include keyboard, mobile, and long-content behavior.
6. Avoid unsupported scanner claims.

Prefer small, reviewable pull requests. UI changes should include before-and-after screenshots and a brief test note.
